# Security Headers Auditor v0.8.0

## Release status

Stable release, published 2026-07-20. The annotated `v0.8.0` tag passed its
tag-triggered CI and release workflow; the published wheel, source distribution,
checksums, and provenance attestation were independently verified. See the
[v0.8 release gate](../RELEASE_GATE_V0.8.md) for the reproducible evidence.

## Portable Review Evidence Capsule

v0.8 adds an offline-verifiable, deterministic review capsule for one current
policy assurance or route assurance result. It binds the canonical scope,
compact `review-json` assessment, optional approved baseline, static profile
definitions, and per-entry SHA-256 manifest without performing another audit.

- Creation and verification never make target or third-party network requests.
- The verifier never extracts content and rejects unsafe ZIP structure,
  malformed/non-canonical JSON, incompatible bindings, and digest mismatches.
- The compact assessment omits raw response-header values, URLs, response
  metadata, diagnostic prose, credentials, cookies, query strings, and fragments.
- The manifest supplies content-integrity evidence only when the reviewer has a
  trusted expected digest; it does not authenticate an author or prove a target
  is secure.

## Boundaries

v0.8 does not add signing, hosted artifact storage, key management, automatic
approval, target discovery, crawling, authentication, fuzzing, browser
automation, exploitation, compliance certification, or vulnerability claims.

Scoring, profiles, CSP parser behavior, target-policy assurance, and framework
evidence mappings remain unchanged, so methodology remains `0.5.0`.

See the [v0.8 release gate](../RELEASE_GATE_V0.8.md) and
[v0.8 specification](../V0.8_METHODOLOGY_SPECIFICATION.md).
