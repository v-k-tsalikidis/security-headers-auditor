# v0.2.0 Release Notes

## Summary

This release turns Security Headers Auditor from a simple header presence checker into a more mature, risk-aware assessment tool.

The focus of v0.2.0 is not feature volume. It is assessment quality: clearer methodology, weighted scoring, better report structure, and public-safe documentation that explains what the tool does and what it deliberately does not claim.

## Added

- Weighted baseline scoring for common browser security headers.
- Finding categories:
  - baseline findings that affect the score;
  - contextual checks that are reported but not scored;
  - information-disclosure observations.
- Weak-value detection for selected headers such as HSTS, CSP, Referrer-Policy, X-Frame-Options, Permissions-Policy, COOP, and CORP.
- Markdown executive summary with severity counts.
- Detailed baseline findings table with evidence, points, and recommendations.
- Contextual checks section for headers that depend on endpoint type and compatibility.
- Information-disclosure observations for selected platform/framework headers.
- Differentiation brief.
- Methodology document with references to OWASP, MDN, Mozilla HTTP Observatory, and web.dev.
- Deterministic sample report.
- GitHub Actions CI workflow for Python 3.10, 3.11, and 3.12.
- Apache-2.0 license.

## Changed

- Replaced flat present/missing scoring with a weighted, risk-aware scoring model.
- Updated README to explain project purpose, differentiation, checked headers, methodology, public safety, and roadmap.
- Expanded tests to cover scoring, weak values, disclosure observations, and Markdown report grouping.

## Security And Safety Notes

This remains a read-only educational and portfolio project. It does not crawl, exploit, brute-force, fuzz, authenticate, bypass controls, or inspect private content.

A strong score does not prove that an application is secure. A weak score means that baseline browser-side hardening should be reviewed.
