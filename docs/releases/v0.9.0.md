# Security Headers Auditor v0.9.0

## Release status

Published and independently verified on 2026-07-20. The annotated `v0.9.0`
tag points to commit `77e997d`; the final release gate records the successful
CI, pre-tag artifact, checksum, provenance, tag-triggered publication, and
published-download verification evidence.

## Workspace Audit History

v0.9 adds a local, bounded history of the last 50 workspace audit sessions.
Each session records its UTC completion time, scope, audit ID, compact target
scores, and outcome. The History view keeps sessions distinct after later
audits; it does not store raw response-header values or full report content.

Every current-run report export now includes the policy scope, UTC timestamp,
and audit-ID suffix in its filename, preventing successive downloads from
overwriting one another.

## Public Operation And Product Identity

The release also clarifies the public operating boundary: Security Headers
Auditor is a local, read-only tool intended for systems the operator owns,
administers, or is expressly authorized to assess. It uses a bounded request
model and does not crawl, authenticate, fuzz, exploit, or bypass controls.

The workspace identifies the independent project and its author, Vasileios
Tsalikidis, through a restrained typographic identity rather than generic
cyber-security iconography. The accompanying product-positioning note explains
the project's profile-aware, reviewable, local-first design without claiming
that it proves a target secure or replaces application security testing.

## Compatibility

Workspace documents migrate deterministically to schema `1.2`. Methodology
remains `0.5.0`; scores, profiles, policy/baseline behavior, framework mappings,
and request boundaries are unchanged. See the
[v0.9 specification](../V0.9_WORKSPACE_AUDIT_HISTORY.md) and
[ADR 0009](../adr/0009-bounded-workspace-audit-history.md).
