# Security Headers Auditor v0.7.0

## Release status

Stable release, published 2026-07-19. The annotated `v0.7.0` tag passed its
tag-triggered CI and release workflow; the published wheel, source distribution,
checksums, and provenance attestation were independently verified. See the
[v0.7 release gate](../RELEASE_GATE_V0.7.md) for the reproducible evidence.

## Controlled Route Assurance

v0.7 adds approved-baseline drift detection for one explicit route-comparison
manifest. A candidate stores only scope, declared profile, score, and
scored-control state. It never includes raw response-header values.

- Create a new candidate with `--write-route-baseline PATH` after a complete
  route-comparison run.
- Review the candidate outside the tool, then enforce it with
  `--route-baseline PATH`.
- Candidate paths are never overwritten, and writing a candidate cannot be
  combined with enforcing a baseline.
- Scope, schema, methodology, mapping set, route ID, path, and profile changes
  require review and re-baselining rather than silent migration.
- Score, status, severity, points, new-actionable-control, and
  missing-control regressions return exit code `1`; invalid configuration or
  operational audit errors return `2`.

## Boundaries

v0.7 does not crawl, discover routes, authenticate, follow cross-origin
redirects, store raw response headers, validate browser runtime behavior, or
make compliance, certification, vulnerability, or effectiveness claims.

The existing profile scoring and framework-evidence mapping set are unchanged,
so methodology remains `0.5.0`.

See the [v0.7 release gate](../RELEASE_GATE_V0.7.md) and
[v0.7 specification](../V0.7_METHODOLOGY_SPECIFICATION.md).
