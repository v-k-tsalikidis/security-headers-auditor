# v0.6.1 Release Notes

Status: **stable when published.** The tag-triggered workflow publishes this
note only after it builds and tests the distributions, generates SHA-256
checksums, and records GitHub artifact provenance.

## Delivery Hardening

- Replaced every GitHub Actions step that declared a Node 20 action runtime
  with reviewed, immutable SHA pins that declare Node 24.
- Replaced `pnpm/action-setup` with Node 24's bundled Corepack. The project and
  workflow both pin and verify `pnpm@11.9.0`, while retaining the pnpm-store
  cache through an immutable GitHub action.

## No Auditor Behavior Change

v0.6.1 retains the v0.6.0 audit engine, CSP assessment behavior, route
comparison, profile export, policies, baselines, schemas, framework mappings,
and methodology `0.5.0`. It is a delivery-control maintenance release, not a
new security finding, policy, or compliance capability.

## Preserved Boundaries

- The tool remains read-only, operator-scoped, and non-crawling. It does not
  authenticate, fuzz, exploit, brute-force, bypass controls, or send results to
  a hosted service.
- Framework mappings remain supporting engineering evidence only. The release
  makes no certification, compliance, endorsement, exploitability, or security
  effectiveness claim.

See the [v0.6.1 release gate](https://github.com/v-k-tsalikidis/security-headers-auditor/blob/v0.6.1/docs/RELEASE_GATE_V0.6.1.md)
for the recorded verification evidence.
