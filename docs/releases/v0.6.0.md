# v0.6.0 Release Notes

Status: **stable when published.** The tag-triggered release workflow publishes
this note only after it builds and tests the distributions, generates SHA-256
checksums, and records GitHub artifact provenance.

## Added

- A deterministic, schema-bound profile-definition export for offline review and
  CI configuration checks. It performs no network request and contains no target,
  response, or raw-header data.
- A bounded route-comparison mode for 2–25 explicit, query-free paths on one
  authorized origin. It does not crawl or discover routes; like-for-like
  differences are review signals rather than vulnerability, policy, or
  compliance decisions.
- A shared CSP parser that retains the first duplicate directive, preserves
  nonce/hash token case, detects valid nonce/hash syntax and `data:` script
  sources, bounds parsing to 16 KiB, and flags malformed or multi-policy input
  for review.

## Methodology Migration

v0.6.0 uses methodology `0.5.0`. Its CSP assessment semantics intentionally
invalidate v0.4 policy and baseline documents: review the changed methodology
and create a new approved baseline. This is not an automatic migration and does
not prove browser enforcement, nonce lifecycle, application compatibility,
document-resource coverage, or bypass resistance.

## Preserved Boundaries

- The tool remains read-only, operator-scoped, and non-crawling. It does not
  authenticate, fuzz, exploit, brute-force, bypass controls, or send results to
  a hosted service.
- Framework mappings remain supporting engineering evidence only. The release
  makes no certification, compliance, endorsement, exploitability, or security
  effectiveness claim.

See the [v0.6 release gate](https://github.com/v-k-tsalikidis/security-headers-auditor/blob/v0.6.0/docs/RELEASE_GATE_V0.6.md)
for the recorded verification evidence.
