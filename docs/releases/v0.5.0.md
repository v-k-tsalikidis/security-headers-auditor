# v0.5.0 Release Notes

Status: **stable when published.** The release workflow publishes this note only
after the distribution build, verification checks, checksums, and provenance
attestation succeed.

## Added

- A loopback-only local workspace for configuring an authorized target
  inventory, running the existing Python audit engine, inspecting results, and
  exporting current evidence.
- Explicit target profiles and assurance expectations in the workspace, with
  disabled targets excluded from single assessment, assurance, and new baseline
  candidates.
- Local SQLite persistence with canonical workspace JSON import/export,
  bounded migration, optimistic revision protection, and deliberate clear-data
  workflows.
- Preview-and-confirm import. Imported targets never run automatically.
- In-memory current-result detail, local summary persistence without raw header
  values, and explicit detailed report export.
- Workspace views for findings, assurance diagnostics, baseline comparison, and
  non-scoring framework evidence with stated limitations.
- A reviewed dependency and license inventory, distributed third-party notices,
  immutable action pins, package cleanliness checks, and a workflow configured
  to produce SHA-256 checksums and GitHub artifact provenance attestation before
  publication.

## Preserved

- The CLI remains the same read-only, operator-scoped assessment engine.
- v0.4 profile scoring, policy-as-code, baseline, regression, output, query
  redaction, and redirect boundary semantics remain intact.
- Framework relationships remain supporting engineering evidence, not
  compliance proof or certification.
- The local workspace has no accounts, analytics, telemetry, cloud sync,
  third-party runtime assets, crawlers, stored credentials, or remote scheduling.

## Verification Boundaries

The workspace browser QA uses deterministic local fixtures and one
Chromium-based browser. It is evidence for the recorded flows, not a statement
of cross-browser, assistive-technology, privacy, or framework certification.

See the [v0.5 release gate](../RELEASE_GATE_V0.5.md) for the evidence required
before this version is marked stable.
