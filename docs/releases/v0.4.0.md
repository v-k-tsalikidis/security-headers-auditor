# v0.4.0 Release Notes

Status: **release candidate**

## Added

- JSON policy-as-code continuous assurance mode.
- Explicit per-target score, severity, required-control, reporting, and isolation expectations.
- Deterministic approved baseline snapshots without raw header values or timestamps.
- Baseline creation restricted to fully passing assurance runs.
- Profile, score, control-state, and new-actionable-finding regression detection.
- Stable CI exit codes for pass, assurance failure, and operational failure.
- Reporting-Endpoints syntax, trustworthiness, and CSP linkage analysis.
- Legacy Report-To and CSP report-uri compatibility observations.
- Cross-origin isolation bundle assessment for coherent COOP/COEP readiness.
- Versioned compliance-evidence manifest with rationale and limitations.
- SARIF 2.1.0 and JUnit XML outputs.
- Human-readable assurance summaries in Markdown and offline HTML.
- JSON Schemas for policy and baseline artifacts.
- Committed, reviewed CI fixture baseline with byte-stable regeneration checks.
- Deterministic localhost CI artifact exercise.

## Preserved

- v0.3 profile taxonomy and 100-point scoring.
- Conservative auto-detection and explicit manual override.
- Query and fragment redaction.
- Cross-origin redirect boundary.
- Script-free, dependency-free HTML report.
- No remote-site dependency in the automated test suite.

## Interpretation

The new evidence mappings and assurance results do not certify compliance. A
reporting pass does not prove report delivery or triage. A cross-origin isolation
pass describes response-level readiness and does not prove runtime state or
resource-graph compatibility.
