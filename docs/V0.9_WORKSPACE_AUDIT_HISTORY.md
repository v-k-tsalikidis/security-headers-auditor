# v0.9 Workspace Audit History Specification

## Purpose

v0.9 makes successive local workspace audits distinguishable. It preserves a
bounded local session trail and makes explicit report downloads unique and
date-bearing, without changing the audit engine or storing raw response headers
as workspace history.

## Session Contract

Every completed **Run audit**, **Run assurance**, or baseline-candidate run
creates one newest-first `audit_history` entry in workspace schema `1.2`.

Each entry contains only:

- UUID `audit_id`;
- UTC `completed_at` timestamp;
- `target` or `assurance` scope;
- policy name, outcome, and stable exit code;
- per-assessment target ID, redacted-safe target, selected profile, score,
  outcome, and exit code.

The history is capped at 50 entries. On entry 51, the oldest entry is removed.
The History view renders the persisted records, not the transient detailed run.

## Export Contract

The report for the current run is explicitly downloaded in the chosen format.
Its filename is:

```text
<policy>-<scope>-<UTC YYYYMMDDTHHMMSSZ>-<audit-id-prefix>-report.<extension>
```

For example:

```text
production-assurance-customer-portal-20260720T103000Z-c8a07661-report.html
```

The timestamp and UUID prefix prevent a later audit from reusing the same
download name.

## Retention Boundary

History never contains raw header values, full findings, detailed error traces,
cookies, credentials, query strings, or report content. Full detailed evidence
is available only while the current workspace process retains the run and in
reports deliberately downloaded by the operator. See
[ADR 0009](adr/0009-bounded-workspace-audit-history.md) and the
[privacy documentation](PRIVACY_ACCESSIBILITY_AUTHORIZATION.md).

## Compatibility And Non-Goals

Existing workspace documents migrate through `1.0 → 1.1 → 1.2` or
`1.1 → 1.2`; the new field starts empty. Existing policies, approved baselines,
profiles, scores, methodology `0.5.0`, framework mappings, request behavior,
and authorization boundaries are unchanged.

v0.9 does not add report archival, remote storage, scheduling, notifications,
crawling, authentication, or a new security/compliance claim.
